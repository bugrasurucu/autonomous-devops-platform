# Contributing to autonomous-devops-platform

Thank you for your interest in contributing! Please take a moment to review these guidelines.

## How to Contribute

1. **Fork** the repository and create a branch from `main`.
2. **Install** dependencies and make sure the project builds locally.
3. **Make your changes** with clear, focused commits.
4. **Test** your changes thoroughly before submitting.
5. **Open a pull request** against `main` and fill in the PR template.

## Reporting Bugs

Use the [Bug Report](.github/ISSUE_TEMPLATE/bug_report.md) issue template and provide as much detail as possible.

## Requesting Features

Use the [Feature Request](.github/ISSUE_TEMPLATE/feature_request.md) issue template to propose new functionality.

## Code Style

- Follow the conventions already present in the codebase.
- Keep commits small and descriptive.
- Write meaningful commit messages.

## Pull Request Guidelines

- One logical change per PR.
- Reference related issues with `Closes #<issue>` where applicable.
- Ensure CI checks pass before requesting review.

## Code of Conduct

This project follows the [Contributor Covenant Code of Conduct](CODE_OF_CONDUCT.md). By participating you agree to abide by its terms.
